# Skin Match and AI Review Backend Specifications

## 1. Skin Match Score

### 1.1 Status Enum
The endpoint `/api/products/:id/skin-match/me` returns `data.status` as one of the following:
- `READY`: Score is calculated and valid.
- `PROFILE_REQUIRED`: The customer has no beauty profile yet.
- `PROFILE_INCOMPLETE`: The customer's profile lacks key information to confidently determine a match (e.g. `confidence_score < 30`).
- `INSUFFICIENT_PRODUCT_DATA`: The product doesn't have enough canonical properties mapped for matching.

### 1.2 Match Level Enum
- `EXCELLENT_MATCH`
- `GOOD_MATCH`
- `MODERATE_MATCH`
- `CAUTION`
- `INSUFFICIENT_DATA`

## 2. AI Review Summary

### 2.1 Status Enum
The endpoint `/api/products/:id/review-insights` returns `data.status` as one of the following:
- `PENDING`: Requested but not generated yet (UI should show "Đang tổng hợp...").
- `GENERATING`: Currently being generated by the worker.
- `READY`: Generated and valid.
- `STALE`: Valid but technically outdated (UI can show it, worker will regenerate in the background).
- `FAILED`: Generation failed after retries.
- `INSUFFICIENT_REVIEWS`: Product has less than 5 reviews.

## 3. Web Worker Details
The `ReviewAiWorker` runs seamlessly behind the scenes querying for `PENDING` and `STALE` items using a MongoDB lock mechanism (`generation_lock_until`). It's configured via `.env` parameter `AI_REVIEW_WORKER_INTERVAL_MS` (default 60000ms).
